## The `account-manager` role — full architecture

```
Subsumptive admin-panel chain:           Parallel scoped roles:
  viewer ← editor ← admin ← owner          sales            free-trial-only
                                           account-manager  non-free-trial-only
                                           customer         portal-only
```

**BE middleware** (`artifacts/api-server/src/middlewares/role-auth.ts`):
- `isAccountManager(req)` — role check.
- `requireSalesAllowed` — now accepts `["sales", "account-manager", "viewer", "editor", "admin", "owner"]`. Both scoped roles share this gate.
- `requireExecutorOrSalesAllowed` — same, plus accepts the executor token.

**BE scope** (`artifacts/api-server/src/lib/scoped-access.ts`):
- `getScopedClientIds(req)` returns `number[]` of eligible client IDs for sales OR account-manager, returns `null` for unscoped sessions.
- `assertScopedAccessToClient(req, res, clientId)` is the inline gate; 404s if outside scope.
- **`sales-scope.ts` is obsolete but still in the tree.** It has zero imports now; safe to delete on the next cleanup pass.

**FE sidebar** (`artifacts/admin-panel/src/components/layout/sidebar.tsx`):
- Flags `salesAllowed` and `accountManagerAllowed`. Both required for items visible to both.
- account-manager sees ONLY Dashboard, Clients, Rankings (Period + Bi-Weekly), Keywords (both children).

**FE route gates** (`artifacts/admin-panel/src/App.tsx`):
- `AdminTierGate` redirects sales + account-manager. Used on `/archived`, `/plans`, `/packages`, `/metrics`, `/sessions/*`, `/keyword-rotation/*`, `/admin/*`.
- `AccountManagerOrAdminGate` redirects sales only. Used on `/keywords`, `/keywords/all`.
- `OwnerOrSalesGate` allows owner + sales. Used on `/sales-ai`.
- `OwnerGate` allows owner only. Used on `/reports/*`, `/admin/variants`.

## Diagnostic queries you'll re-run

**Stale ranking count (broken down by plan):**
```sql
WITH latest AS (
  SELECT k.id, COALESCE(cap.plan_type, '(no plan)') AS plan_type,
         MAX(rr.date::date) AS last_audit
  FROM keywords k
  JOIN clients c ON c.id = k.client_id
  LEFT JOIN client_aeo_plans cap ON cap.client_id = k.client_id
  LEFT JOIN ranking_reports rr ON rr.keyword_id = k.id AND rr.status = 'success'
  WHERE k.is_active = true AND k.archived_at IS NULL
    AND c.status = 'active' AND c.archived_at IS NULL
  GROUP BY k.id, cap.plan_type
)
SELECT plan_type, COUNT(*) AS total,
       COUNT(*) FILTER (WHERE last_audit IS NULL) AS never,
       COUNT(*) FILTER (WHERE last_audit < CURRENT_DATE - 14) AS stale_14d
FROM latest GROUP BY plan_type ORDER BY total DESC;
```
