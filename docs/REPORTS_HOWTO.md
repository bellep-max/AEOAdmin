# Generating Reports — What to Pick & What to Search

A practical guide for the team: how to pull the right rankings report, what each
filter does, and how to read/export it. Pair this with the client guide
([CLIENT_GUIDE_NONTECH.md](./CLIENT_GUIDE_NONTECH.md)).

---

## Where to go

**Rankings** (sidebar → Analytics → Rankings) has two views:

1. **Period Comparison** — current vs. previous, per keyword × platform. Use for
   "how did we move this cycle?"
2. **Bi-Weekly Report** — the formal client deliverable for one bi-weekly batch.

---

## Step 1 — Narrow to who the report is for

Use the three dropdowns (left to right, each narrows the next):

| Filter       | Pick this when…                                                                  |
| ------------ | -------------------------------------------------------------------------------- |
| **Client**   | You want one client's report. Leave "All Clients" only for an internal overview. |
| **Business** | A client has multiple locations/businesses and you want just one.                |
| **Campaign** | You want a single campaign (plan) within that business.                          |

> Rule of thumb for a **client-facing report**: always set **Client** (and Business
> if they have more than one). "All Clients" is for internal review only.

---

## Step 2 — Choose the time frame

**Period Comparison** dropdown:

| Option                     | What "current vs previous" means                                                               |
| -------------------------- | ---------------------------------------------------------------------------------------------- |
| **Biweekly**               | Latest run vs the run before it (the normal cadence). **Default for client reports.**          |
| **Monthly**                | This month's latest vs last month's.                                                           |
| **Quarterly**              | This quarter vs last quarter.                                                                  |
| **Since start (lifetime)** | Most recent vs the very first ranking — the full journey. Great for "look how far we've come." |

The header always tells you the exact dates it's comparing, e.g.
_"Current: Jun 2 – Jun 17 · Compared with: May 19 – Jun 2."_ **Read that line before
exporting** so you know what the client will see.

### Pinning exact dates (optional)

The **FIRST / PREV / CURRENT** date pickers let you force a specific run date for a
column instead of "whichever was latest." Use this only when you need an exact
historical snapshot (e.g. recreate a report from a past date). Leave them on
**Any** for normal reporting.

---

## Step 3 — Search / focus (optional)

- **Search keywords box** — type part of a keyword to filter the list (e.g.
  "weight loss"). Good for spot-checking one phrase live with a client.
- **All audits** dropdown — limit to a particular audit run if needed.
- The colored summary chips (↑ improved / ↓ declined / = steady / + new) tell you
  the cycle's story at a glance before you scroll.

---

## Step 4 — What to actually look at (the "what to search" answer)

When generating or reviewing a report, scan for these, in order:

1. **Top 3 count** — how many keywords are in positions 1–3 right now. This is the
   headline metric.
2. **Improved (↑)** — wins to highlight to the client this cycle.
3. **Declined (↓)** — anything that slipped; have an explanation ready (these
   become the next optimization focus).
4. **New** — recently added keywords getting their first data.
5. **Won / Locked** — keywords that reached and held Top 3 (the success stories).
6. **Dates** — confirm "Current" is the **newest** date and "Last 2 weeks" is the
   prior one. If they ever look reversed, flag it — don't send.

---

## Step 5 — Export

| Button          | Use for                                                      |
| --------------- | ------------------------------------------------------------ |
| **CSV**         | Raw data for spreadsheets / further analysis.                |
| **PDF**         | The polished, client-ready document.                         |
| **Send Report** | Email it directly to the client (check the recipient first). |

Every exported rank is backed by a **saved screenshot** of the real AI answer on
that date — so any number can be proven if a client asks.

---

## Quick recipe: a standard client report

1. Client = _(the client)_; Business = _(if multiple)_.
2. Period = **Biweekly**.
3. Confirm the date line at the top reads the two most recent runs.
4. Glance at the chips: Top-3 count, ↑ wins, ↓ to explain.
5. Export **PDF** → review → **Send Report**.

That's it — same five steps every time.
